# Code Review

## Overview

Review code for spec compliance and code quality with objective, evidence-based findings.

**Announce at start:** `Using the code-review skill for spec-compliance and code-quality review.`

## The Iron Law

```
REVIEW MUST BE OBJECTIVE AND EVIDENCE-BASED
```

Never make subjective claims without evidence. Always cite specific code locations.

## When to Use

Use `code-review` when:
- Before merging code
- After implementation
- During code review phase

## Relation to Workflows

- Used in workflow Step 6 (REVIEW)
- Three-stage review: spec compliance, code quality, human verdict
- Architect is requester, Reviewer agents return findings
